# Statecraft 项目进度总结

**版本**：v0.1.0  
**最后更新**：2025-03-18

---

## 1. 项目概览

Statecraft 是一款基于 Godot 4.6.1 的卡片对决游戏，使用 GDScript 开发。支持论策（单机 vs AI）与演兵（局域网 2 人对战）两种模式。

---

## 2. 当前进度

### 2.1 已完成功能

| 模块 | 功能 | 状态 |
|------|------|------|
| **主菜单** | 开始游戏、论策/演兵模式选择、设置、Mod 管理 | ✅ |
| **战斗核心** | 抽牌、回合切换、同名矛盾、分数超限矛盾、结算 | ✅ |
| **卡牌系统** | 空牌、见好就收、一石二鸟、Mod 注册卡牌 | ✅ |
| **联机** | 主机创建/加入、Mod 校验、抽牌/结算 RPC、收牌动画同步 | ✅ |
| **他山之石 Mod** | 抽到时选择一石二鸟+对手牌移除、联机客户端选择、特效同步 | ✅ |
| **UI** | 手牌/回合区显示、抽牌动画、矛盾淡出、他山之石原位特效 | ✅ |

### 2.2 游戏模式

- **论策**：单机，玩家 vs AI，AI 自动抽牌/结束回合
- **演兵**：局域网 2 人对战，主机权威，客户端 RPC 请求

### 2.3 核心架构

```
主菜单(main_menu) → 模式选择 → 对决(battle_test)
                    ↓
        CardManager / BattleManager / NetworkManager
                    ↓
        ModLoader 触发钩子 → Mod 脚本(如他山之石)
```

---

## 3. 技术索引

### 3.1 核心脚本

| 脚本 | 路径 | 职责 |
|------|------|------|
| main_menu | scenes/main/main_menu.gd | 主菜单、模式选择、演兵对话框、Mod 面板 |
| battle_test | scenes/battle/battle_test.gd | 战斗 UI、手牌/回合区、抽牌动画、矛盾特效 |
| card_ui | scenes/ui/card_ui.gd | 卡牌 UI 组件 |
| CardManager | scripts/core/card_manager.gd | 牌堆、抽牌、矛盾检测、回合结算 |
| BattleManager | scripts/core/battle_manager.gd | 状态机、回合、RPC、Mod 劫持 |
| NetworkManager | scripts/core/network_manager.gd | ENet 联机、场景同步、Mod 校验 |
| ModLoader | scripts/core/mod_loader.gd | Mod 钩子、API 调用 |
| ModManager | scripts/core/mod_manager.gd | Mod 加载、mod_info、覆盖检测 |
| GameLogger | scripts/autoload/game_logger.gd | 日志缓冲、每局保存 |
| CardResource | scripts/data/card_resource.gd | 卡牌数据定义 |
| 他山之石 | mods/tashanzhishi_mod/main.gd | 抽到时选择移除一石二鸟+对手牌 |

### 3.2 关键流程

| 流程 | 说明 |
|------|------|
| 抽牌 | UI 请求 → BattleManager.request_draw → CardManager.draw_card → post_draw_card 钩子 → card_drawn 信号 → battle_test 动画 |
| 联机抽牌 | 客户端 RPC → 主机 _execute_draw → _rpc_apply_draw 下发 → 客户端 apply_draw_from_data |
| 他山之石联机 | 主机 post_draw_card 通知客户端 → _rpc_tashanzhishi_mode_started → 客户端 sync_tashanzhishi_mode + post_draw_animation → 选择后 request_tashanzhishi_selection RPC → 主机 _apply_tashanzhishi_selection → _rpc_mod_soft_remove_effect 同步特效 |
