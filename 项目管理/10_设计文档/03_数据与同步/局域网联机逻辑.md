# 局域网联机逻辑设计文档

**版本**：v0.1.1-1  
**最后更新**：2026-03-28

---

## 1. 架构概览

### 1.1 核心原则

- **主机权威（Server Authoritative）**：所有游戏逻辑计算仅在主机执行，客户端仅发送输入、接收结果并渲染
- **镜像 UI**：双方屏幕布局镜像，主机手牌在主机底部、客户端手牌在客户端底部；对方手牌显示在顶部
- **一退全退**：任意一方点击「返回」时，双方同步退出到主菜单并断开连接

### 1.2 角色映射

| 角色 | player_index | peer_id | 说明 |
|------|--------------|---------|------|
| 主机 | 0 | 1 | 创建房间，执行所有计算 |
| 客户端 | 1 | 2 | 加入房间，仅发送输入、接收渲染数据 |

### 1.3 数据流

```
客户端点击 → RPC 请求 → 主机计算 → RPC 下发结果 → 客户端渲染
```

### 1.4 渠道权限对齐

- Android / TapTap 渠道若保留局域网联机能力，提审前需在 TapTap 后台权限说明中勾选“网络权限”
- 权限用途说明统一填写：`用于局域网对决数据同步`
- 该项与 `scripts/core/network_manager.gd`、`scripts/core/battle_manager.gd` 的局域网对战能力保持一致，功能下线时需同步检查后台权限说明与协议文案

## 2. NetworkManager（网络管理器）

**路径**：`scripts/core/network_manager.gd`  
**Autoload**：`/root/NetworkManager`

### 2.1 连接管理

| 方法 | 说明 |
|------|------|
| `create_game(port)` | 主机创建房间，`max_clients=1`，仅支持 2 人对战 |
| `join_game(ip, port)` | 客户端加入房间 |
| `disconnect_peer()` | 断开连接并重置状态 |

### 2.2 身份与判定

| 方法/属性 | 说明 |
|-----------|------|
| `is_host` | true=主机，false=客户端 |
| `get_local_player_index()` | 主机=0，客户端=1；单机恒为 0 |
| `is_my_turn(current_index)` | 判定 current_index 是否为本地玩家回合 |
| `is_multiplayer()` | 是否处于联机模式 |

### 2.3 场景同步

- **主机**：`_on_peer_connected` 时调用 `_sync_to_battle()`，生成随机种子，重置 Manager，RPC 广播后跳转对决场景
- **客户端**：`_on_connected_to_server` 时等待；收到 `_rpc_sync_to_battle` 后同步种子、重置 Manager、跳转对决场景

### 2.4 退出与重新开始同步

| RPC | 说明 |
|-----|------|
| `_rpc_exit_to_main` | `@rpc("any_peer", "call_local")`，任意一方调用后双方执行 `disconnect_peer()` 并跳转主菜单 |
| `_rpc_restart_game` | `@rpc("authority")`，主机调用 `request_restart_game()` 后广播，双方同步重置 Manager 并重载对决场景 |

| 方法 | 说明 |
|------|------|
| `request_exit_game()` | 一退全退，RPC 广播后双方退出 |
| `request_restart_game()` | 仅主机可触发；单机直接重置重载；联机时 RPC 广播后双方同步重置重载 |

## 3. BattleManager（战斗管理器）

**路径**：`scripts/core/battle_manager.gd`  
**Autoload**：`/root/BattleManager`

### 3.1 回合权威

- **`current_player_index`**：仅主机通过 RPC 修改，客户端绝不自行计算
- **`next_turn()`**：主机计算新索引后调用 `_rpc_sync_turn.rpc(new_index)`；单机时本地直接执行

### 3.2 抽牌流程

| 步骤 | 主机 | 客户端 |
|------|------|--------|
| 1 | UI 调用 `request_draw()` | UI 调用 `request_draw()` → `_rpc_request_draw.rpc_id(1)` |
| 2 | `_execute_draw()`：`cm.draw_card()` 计算 | — |
| 3 | 构建 `card_data` Dictionary（name, score, ability_id 等） | — |
| 4 | `_rpc_apply_draw.rpc(player_index, card_data, deck_count)` | — |
| 5 | — | `cm.apply_draw_from_data()` 创建卡牌副本，不触碰 global_deck |
| 6 | — | `card_drawn.emit()` 触发 UI 显示 |

### 3.3 结束回合流程

| 步骤 | 主机 | 客户端 |
|------|------|--------|
| 1 | `_execute_end_turn()`：捕获 turn_container 卡牌数据 | `_rpc_request_end_turn.rpc_id(1)` |
| 2 | `cm.settle_turn()` 执行结算（含 Mod 钩子） | — |
| 3 | `_rpc_apply_settlement_result.rpc(...)` | — |
| 4 | `_rpc_sync_hand_collection.rpc(player_who_ended)` | — |
| 5 | `_apply_turn_switch(next_index)` | 收到 apply → `cm.apply_settlement_result()` |
| 6 | — | 收到 sync_hand → `hand_collection_requested.emit()` |
| 7 | — | `_apply_turn_switch(next_index)` |

### 3.4 强制结束（见好就收等）

| 步骤 | 主机 | 客户端 |
|------|------|--------|
| 1 | `force_end_turn` 信号 → `_on_force_end_turn()` | —（客户端不收到此信号） |
| 2 | `_execute_force_settle_and_next()`：结算、捕获数据 | — |
| 3 | `_rpc_force_settle_and_next.rpc(...)` | — |
| 4 | `_rpc_sync_hand_collection.rpc(...)` | — |
| 5 | `_apply_turn_switch(next_index)` | 收到 RPC → 应用数据、切换回合 |

### 3.5 矛盾同步

| 场景 | RPC | 说明 |
|------|-----|------|
| 同名矛盾 | `_rpc_contradiction_triggered` | 主机在 `_transition_to(CONTRADICTION)` 时广播，客户端清空 turn_container、执行矛盾 UI |
| 分数超限 | `_rpc_sync_scores` + `_rpc_contradiction_triggered` | 主机先回滚分数并 RPC，再过渡到 CONTRADICTION |

### 3.6 RPC 汇总

| RPC | 权限 | 说明 |
|-----|------|------|
| `_rpc_sync_to_battle` | authority | 场景跳转 + 种子同步 |
| `_rpc_exit_to_main` | any_peer, call_local | 一退全退 |
| `_rpc_restart_game` | authority | 重新开始（主机广播，双方同步重置重载） |
| `_rpc_request_draw` | any_peer | 客户端请求抽牌 |
| `_rpc_apply_draw` | authority | 主机下发卡牌数据 |
| `_rpc_request_end_turn` | any_peer | 客户端请求结束回合 |
| `_rpc_request_empty_turn` | any_peer | 空回合结束 |
| `_rpc_apply_settlement_result` | authority | 结算结果（卡牌、分数、下一回合） |
| `_rpc_sync_hand_collection` | authority, call_local | 收牌动画同步 |
| `_rpc_sync_turn` | authority, call_local | 回合切换 |
| `_rpc_sync_scores` | authority | 分数同步 |
| `_rpc_contradiction_triggered` | authority | 矛盾触发 |
| `_rpc_game_ended` | authority | 游戏结束 |
| `_rpc_force_settle_and_next` | authority | 强制结算（见好就收） |
| `_rpc_apply_mod_settlement` | authority | Mod 额外行动结算 |
| `_rpc_update_ui_state` | authority | 回合 Label 刷新 |
| `_rpc_lock_ui` | any_peer | 神来之笔选择时锁定对手 UI |

## 4. CardManager（卡牌管理器）

**路径**：`scripts/core/card_manager.gd`  
**Autoload**：`/root/CardManager`

### 4.1 主机专用

| 方法 | 说明 |
|------|------|
| `draw_card(player_index)` | 从 global_deck 抽牌，返回 `[deck_index, card]` |
| `settle_turn(player_index)` | 计算总分、移入 player_hands、触发 Mod 钩子 |

### 4.2 客户端专用（禁止计算）

| 方法 | 说明 |
|------|------|
| `apply_draw_from_data(player_index, card_data, deck_count)` | 根据主机 Dictionary 创建 CardResource，加入 turn_container，发出 card_drawn |
| `apply_settlement_result(player_who_ended, cards_data, score_0, score_1)` | 根据主机数据创建卡牌并加入 player_hands，清空 turn_container |

### 4.3 卡牌数据 Dictionary

```gdscript
{
    "card_name": String,
    "effect_value": int,
    "ability_id": String,
    "count_towards_contradiction": bool,
    "force_ends_turn": bool
}
```

## 5. battle_test（战斗 UI）

**路径**：`scenes/battle/battle_test.gd`

### 5.1 镜像 UI 映射

| 函数 | 说明 |
|------|------|
| `get_visual_target(logic_player_index)` | logic_player_index == 本地索引 → 底部手牌容器；否则 → 顶部对手容器 |
| `get_target_container(logic_player_index)` | 调用 get_visual_target |

### 5.2 收牌动画

- **单机**：`hand_updated` 信号 → `_animate_cards_to_hand(player_index)`
- **联机**：`hand_collection_requested` 信号 → `_animate_cards_to_hand(logic_player_index)`，`hand_updated` 被忽略

### 5.3 权限限制

| 元素 | 联机客户端行为 |
|------|----------------|
| RestartBtn | `disabled = not multiplayer.is_server()` |
| 设置面板「重新开始」 | 同上 |
| 抽牌/结束回合 | 仅 `is_my_turn` 时可用 |

### 5.4 回合 Label

- 联机：`"我的回合"` / `"对手回合"`（根据 `NetworkManager.is_my_turn(current_player_index)`）
- 单机：`"当前轮到：玩家"` / `"当前轮到：对手"`

## 6. main_menu（主菜单）

**路径**：`scenes/main/main_menu.gd`

### 6.1 演兵入口

- 点击「演兵」→ `_show_yanbing_network_dialog()` 弹出局域网对话框
- 输入 IP、端口，点击「创建房间」或「加入房间」
- 连接成功后，`peer_connected` 触发主机跳转；`connection_succeeded` 触发客户端等待

### 6.2 返回逻辑

- 对决场景设置面板「返回」：联机时调用 `NetworkManager.request_exit_game()`，单机时直接跳转主菜单

---

## 7. Mod 联机约束

### 7.1 主机独占

- `post_draw_animation` 钩子：仅主机执行，客户端 `draw_animation_finished` 直接 `_transition_to` 后返回
- `mod_settle_and_continue`：仅主机执行，客户端调用时直接 return

### 7.2 神来之笔

- 选择时：`request_lock_opponent_ui(true)` → `_rpc_lock_ui.rpc(true)`，对手端禁用抽牌/结束回合
- 选择完成：`request_lock_opponent_ui(false)` 解锁

## 8. 技术索引

### 8.1 核心脚本

| 脚本 | 路径 | 职责 |
|------|------|------|
| NetworkManager | scripts/core/network_manager.gd | 连接、身份、场景同步、退出 |

| CardManager | scripts/core/card_manager.gd | 牌堆、抽牌计算、结算计算、客户端数据应用 |

### 8.2 关键方法

| 方法 | 所属 | 说明 |
|------|------|------|
| get_local_player_index | NetworkManager | 本地玩家索引 |
| is_my_turn | NetworkManager | 回合判定 |

| request_end_turn / _execute_end_turn | BattleManager | 结束回合 |

| 名称 | 说明 |
|------|------|

### 8.4 常量
